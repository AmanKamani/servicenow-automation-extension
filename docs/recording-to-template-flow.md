# Recording to Template/Flow

## Problem Statement

Template/flow creation is currently manual and requires users to understand field keys, label matching, field types, waits, and action ordering. This increases setup time and creates adoption friction for new users.

Pain points:

- Building a complete template from scratch is time-consuming.
- Non-technical users may struggle with selector and wait tuning.
- Recreating similar workflows across environments is repetitive.

## Proposed Design

Add a user-interaction recorder that captures clicks, typing, selections, and navigations, then converts them into a draft template/flow for user review.

### 1) Recorder Session Model

Introduce recording sessions with:

- start/stop controls
- captured events timeline
- target tab association
- session metadata (domain, timestamps)

### 2) Event Capture Pipeline

Capture high-signal events:

- text input changes
- select/choice changes
- checkbox/switch toggles
- button clicks
- URL transitions/navigation points
- dialog interactions where applicable

For each event, derive candidate config:

- suggested `fieldType`
- suggested `displayName`
- suggested `labelMatch`
- inferred action vs input ordering

### 3) Draft Generation

Convert recorded timeline into draft artifacts:

- **Draft Template** for single-page runs
- **Draft Flow** for multi-step/multi-URL runs

Generated output should be editable before save.

### 4) Review and Confirmation UX

Before persisting:

- show generated fields/steps with confidence hints
- allow quick corrections (rename key, adjust labelMatch, change fieldType)
- preview payload schema and default values

## Risks and Constraints

- **Noisy signals:** raw DOM events can over-capture irrelevant actions.
- **Selector quality:** autogenerated label matches may be ambiguous.
- **App-specific widgets:** custom components may need widget-specific heuristics.
- **Privacy/security:** recording may capture sensitive input unless masked.
- **Performance:** event instrumentation must avoid degrading page responsiveness.

## Phased Rollout

### Phase 1 — Manual-Assisted Recorder (MVP)

- Add start/stop recording for one tab.
- Capture core inputs + button clicks.
- Generate draft template only.

### Phase 2 — Flow-Aware Recorder

- Capture navigation boundaries and step transitions.
- Generate draft flows with ordered template chunks.
- Add confidence indicators per generated step.

### Phase 3 — Smart Suggestions and Cleanup

- Deduplicate noisy events.
- Improve key naming and labelMatch suggestions.
- Add privacy controls (field masking and exclusion rules).

## Open Questions

- Should recorder run from sidepanel, options page, or both?
- What fields should be masked by default (passwords, tokens, long text)?
- Should generated drafts be auto-saved or require explicit “Create Draft” action?

## Success Criteria

- Users can produce a usable draft template/flow in minutes instead of manual full setup.
- Generated configs require minimal edits for common forms.
- Recorder remains optional and does not affect existing manual configuration workflows.
